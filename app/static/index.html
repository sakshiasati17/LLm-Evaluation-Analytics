Features
Everything AI teams need
to ship reliably

SentinelAI is the testing and observability layer that sits above your LLMs, the same way unit tests and APM sit above your application code.

🧠
LLM-as-Judge Scoring
Uses a second AI model to semantically score responses, the same approach used by OpenAI Evals, LangSmith, and Braintrust. Exact-match for multiple-choice, judge for open-ended, Jaccard heuristic for offline use.
✅
CI/CD Quality Gates
GitHub Actions integration that blocks pull requests if accuracy drops below threshold or hallucination exceeds the limit. Configurable per-project with min_accuracy, max_hallucination, max_latency, and max_cost.
📊
Real-Time Dashboard
WebSocket-powered dashboard with 6 Chart.js visualizations , radar charts, accuracy trends, hallucination risk, latency comparison, cost distribution, and a time-series drift monitor.
⚡
Async Parallel Evaluation
All evaluation cases run concurrently via asyncio.gather(). Per-case error isolation means if one LLM call fails, the remaining 24 still complete and the failure is recorded separately.
🔀
Multi-Model Comparison
Evaluate 5 providers simultaneously — OpenAI, Anthropic, Google, Cohere, and OpenRouter — and get ranked comparison data sorted by accuracy, latency, and cost in a single API call.
🛡️
Safety Scoring
19-term weighted safety scorer with severity levels for PII leakage, harm facilitation, CSAM, self-harm, hate speech, and attack vectors. Independent from accuracy — surfaces harmful outputs automatically.
📈
Drift Detection
Historical analytics aggregated by model, prompt version, and dataset version. Time-series accuracy trend shows whether AI quality is improving or degrading across deployments over time.
🎯
Standard Benchmarks
MMLU (57-domain knowledge), TruthfulQA (hallucination detection), Reasoning (logic & math), and Coding (Python algorithms) 25 cases each, runnable against any provider in one click.
🔌
Adapter Pattern
Every LLM provider implements a single BaseAdapter abstract class. Adding a new provider means writing one class the evaluator, metrics, and dashboard work with zero changes.
How It Works
From question to insight
in milliseconds
1
Define Test Cases
Submit evaluation cases via API or load one of the 4 built-in benchmarks. Each case has a question, optional reference answer, and metadata.
2
Run Evaluation
The evaluator calls the LLM adapter concurrently for all cases. Each case is isolated, failures are recorded, not propagated.
3
Score Responses
3-tier scoring: exact letter match for MCQ, LLM-as-judge for open-ended, Jaccard heuristic for offline. Safety and hallucination scores computed independently.
4
Gate or Monitor
Results feed the CI/CD gate (blocks bad merges) and the analytics store (builds drift history). Dashboard updates in real time via WebSocket push.
